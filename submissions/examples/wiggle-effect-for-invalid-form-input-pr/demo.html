How It Works

The wiggle animation is triggered using CSS :invalid pseudo-class combined with the :not(:placeholder-shown) selector to ensure the animation only plays after user interaction.

@keyframes wiggleInput {
    0%, 100% { transform: translateX(0); }
    20%      { transform: translateX(-6px); }
    40%      { transform: translateX(6px); }
    60%      { transform: translateX(-4px); }
    80%      { transform: translateX(4px); }
}

input:invalid:not(:placeholder-shown) {
    animation: wiggleInput 0.5s ease-in-out;
    border-color: var(--color-error);
}

Key Features

  • Pure CSS - No JavaScript required for the animation
  • Accessible - Respects prefers-reduced-motion, includes ARIA live regions
  • Responsive - Works on all screen sizes
  • Semantic HTML - Uses proper form elements and labels
  • No Layout Shift - Animation uses transform, doesn't affect layout
  • Customizable - Easy to adjust timing, distance, and colors
